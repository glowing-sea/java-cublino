## Tutor Comment

Top notch submission! It's great to see that you got the assignment working so well. Your
code base is well designed and the GUI is super intuitive and easy to use.

You have effectively used many object oriented features throughout the project,
and this makes your code concise and readable. Your classes all have clear and
distinct purposes. This makes it easy to maintain and update your code. The descriptive
comments also improve the readability of your code, although comments above functions
should be JavaDoc format, which makes the input and output clearer.

Your coding style
throughout the project is generally good. Some of your lines are very long, which impedes
readability (such as in the Board class). Lines longer than 100 characters
should be broken up or wrapped around to avoid this.

You do have some repetitive code, or sections where you have hard-coded many options. For
example, the updateDices function in the Board class and the tip function in the Dice class.
By using logic rather than hardcoding these options you would make your code more readable and
easier to debug.

The two levels of AI work well, and you used your class design to make changing
between these levels very smooth. It's good to see that rather than rewriting the same
code for both game modes you used different heuristics with the same algorithms.
Your heuristic function
shows that you thought about many different
factors that might be advantageous in the different game modes.

You have written some good tests, although a few are testing functions that were already tested
by us and there are some key functions that are missing tests, such as your greedy/minimax
functions. By testing a wider range of functions
it would be easier to check that your program is working as expected.

You have used Git well throughout the assignment with helpful and descriptive
commit messages. It's great to see consistent commits from all team members, showing
good collaboration skills. Most files have authorship details, although some are missing
(e.g. MoveBuilder, Game).

Your GUI is very user-friendly. The net showing the layout of the dice is a nice enhancement to the game and
works really smoothly. The instructions are also very helpful for new users.
Congratulations on completing a really excellent assignment!

+     * Give a Pur or Contra state and a difficulty value n, find the best move.
+     * If n = 1, implement greedy AI. If n = 2 implement Minimax AI.
+     *
+     * How does the function work intuitively?
+     *
+     * The function generate a list of legal moves. For each move, it applies the move and gets a new state resulting
+     * from the move. It then evaluates the new state and save the move and the new state together in a pair called

+    // (By Rajin & Haoting)
+    public static Move bestMove(State state, int difficulty){
+        if (difficulty != 1 && difficulty != 2) throw new IllegalArgumentException();
+        boolean useMinimax = difficulty == 2;
+
+        ArrayList<MovePlus> legalMoves = new ArrayList<>(); // A list of legal moves.
+
+        for (Move move : state.legalMoves()){
+            MovePlus st = new MovePlus(move, state, useMinimax);
+            legalMoves.add(st);
+        }
+        return Collections.max(legalMoves).getMOVE(); // Find the move leading to the state with the highest score.
     }

+    // A pair that store a move and the score of the new state result from the move.
+    private static class MovePlus implements Comparable<MovePlus>{
+        private final Move MOVE; // A move
+        private final int SCORE; // The score of the new state result from the move.
+
+        public MovePlus(Move move, State parent, boolean useMinimax) {
+            this.MOVE = move;
+            boolean currentPlayer = parent.getPlayerTurn(); // The current player who is going to make a move decision.
+            State child = parent.copy(); // Make a child by copying the parent
+            child.applyMove(move); // Apply the move to the child
+
+            if (useMinimax){
+                GameTree tree = new GameTree(child, 3); // Build a tree from the child.
+                this.SCORE = tree.miniMaxAB(-9999, 9999, currentPlayer); } // Evaluate the tree.
+            else
+                this.SCORE = child.stateEvaluate(currentPlayer); // Evaluate the child state directly.
+        }
+        @Override
+        public int compareTo(MovePlus other) {
+            return this.SCORE - other.SCORE; // The best move is the one with the highest score.
+        }
+        public Move getMOVE() { return MOVE; }
+    }
